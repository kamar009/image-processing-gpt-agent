## 1. Продукт и репозиторий

- **Название:** Site image processing MVP (внутреннее имя: **image-processing-gpt-agent**).
- **GitHub:** `kamar009/image-processing-gpt-agent`.
- **Суть:** сервис подготовки изображений для сайта: опциональный **Vision** (несколько провайдеров: OpenAI, Sber, Yandex и др.), пресеты **`product`**, **`category`**, **`banner`**, **`portfolio_interior`**, **`furniture_portfolio`**, валидация, WebP/JPEG/PNG. - **Стек:** Python 3.11+ (CI), FastAPI (`main.py`), Docker Compose (локально + `deploy/sweb` для VPS), GitHub Actions: **CI** + **Deploy SWEB** (SSH на VPS).

- Тип **`furniture_portfolio`** реализован в **main** (API, пайплайн, отдельный Vision-промпт, поля ответа, форма **`/ui`**). Контракт: [docs/FURNITURE_PORTFOLIO_API.md](../docs/FURNITURE_PORTFOLIO_API.md).

## 3. Инфраструктура и прод (SpaceWeb VPS)

- **VPS:** Ubuntu 24.04, в переписке фигурировали IP `77.222.63.137`, API-хост `api.kimoffice9-18.ru`.
- **Путь приложения на сервере:** `/opt/app` (клон репозитория).
- **Пользователь для `git pull` и владелец приложения:** **`deploy`** (не полагаться на `ops`, если нет прав на `.git`). Локальный SSH-ключ разработчика/агента: **`~/.ssh/sweb_deploy_github2`**. Правило в репозитории: [`.cursor/rules/sweb-ssh-deploy.mdc`](../.cursor/rules/sweb-ssh-deploy.mdc).
- **Bootstrap:** `deploy/sweb/bootstrap-vps.sh` (Docker, UFW, fail2ban).
- **Деплой из GitHub:** [`.github/workflows/deploy-sweb.yml`](../.github/workflows/deploy-sweb.yml) — SSH, проверка `.env`, `git pull`, `docker compose -f deploy/sweb/docker-compose.yml build/up`, `sync_internal_presets` в контейнере `api`.
- **Секреты GitHub:** `SWEB_HOST`, `SWEB_USER`, `SWEB_SSH_KEY`, опционально `SWEB_PUBLIC_BASE`.

### Уже выясненные проблемы и решения

- **SSH `publickey` / `ROOT LOGIN REFUSED`:** на VPS в **`/etc/ssh/sshd_config.d/99-hardening.conf`** могло быть `PermitRootLogin no`; для деплоя от root — согласовать политику или использовать **`deploy`**.
- **`dubious ownership` при `git pull`:** `git config --global --add safe.directory /opt/app` для пользователя, от которого тянут репозиторий.
- **VNC без вставки из буфера:** ключ через панель SpaceWeb или `authorized_keys`.
- **Deploy падал на отсутствии `/opt/app`:** первичный clone + bootstrap; затем **`.env`** в `/opt/app` (скрипт `deploy/sweb/init-env-from-example.sh`).
- **Windows PowerShell `ssh-keygen -N ""`:** надёжнее интерактивно два раза Enter или `cmd /c "ssh-keygen ... -N \"\""`.
- **Конфликт имён контейнеров Docker** после переименования compose: `docker rm -f` старых имён + `docker compose … up -d --remove-orphans`.

### Nginx / SSL (`deploy/sweb`)

- До выпуска серта nginx может не подниматься; смок внутри контейнера API: см. [docs/SWEB_RUNBOOK.md](../docs/SWEB_RUNBOOK.md).

## 5. Этап 1 — модуль 2 на ноутбуке (публичный API)

**Цель:** убедиться, что без internal-режима работают health, metrics, process-image, скачивание, UI.

- **Переменные:** `INTERNAL_MODE=0`, при необходимости ключи Vision в `.env` (неверный ключ → ошибки провайдера / fallback — смотреть логи и поля `vision_fallback_*` в ответе).
- **Зависимости тестов:** `pip install -r requirements-dev.txt`.
- **Pytest (примеры):** `tests/test_api_smoke.py`, `test_metrics_disk.py`, `test_upload_limit.py`, `test_all_image_types.py`, `test_furniture_portfolio_api.py`, `test_analyze_furniture_prompt.py`, `test_presets.py`.
- **Ручной запуск:** `python -m uvicorn main:app --host 127.0.0.1 --port 8000`.
- **Smoke против поднятого сервера:** `python scripts/integration_smoke.py --image path/to/file.png --strict` и опционально `--include-furniture`.
