## 3. Инфраструктура и прод (SpaceWeb VPS)

- **VPS:** Ubuntu 24.04, публичный IP (в переписке фигурировал `77.222.63.137`), хост `kamar009r2-vps-1`.
- **Путь приложения на сервере:** `/opt/app` (clone репозитория).
- **Bootstrap:** `deploy/sweb/bootstrap-vps.sh` (Docker, пользователь `deploy`, UFW, fail2ban).
- **Деплой из GitHub:** workflow [`.github/workflows/deploy-sweb.yml`](../.github/workflows/deploy-sweb.yml) — SSH, проверка `.env`, `git config safe.directory /opt/app`, `git pull`, `docker compose -f deploy/sweb/docker-compose.yml build/up`, `sync_internal_presets`.
- **Секреты GitHub:** `SWEB_HOST`, `SWEB_USER` (часто `root`), `SWEB_SSH_KEY`, опционально `SWEB_PUBLIC_BASE`.

### Уже выясненные проблемы и решения

- **SSH `publickey` / `ROOT LOGIN REFUSED`:** на VPS в **`/etc/ssh/sshd_config.d/99-hardening.conf`** было `PermitRootLogin no`, перекрывающее основной `sshd_config`; исправлено на `prohibit-password` (или эквивалент). Пароль по SSH отключён (`PasswordAuthentication no`).
- **`dubious ownership` при `git pull` от root:** `git config --global --add safe.directory /opt/app`.
- **VNC без вставки из буфера:** ключ можно добавить через панель SpaceWeb SSH-ключей или вручную в `authorized_keys`.
- **Deploy падал на отсутствии `/opt/app`:** первичный clone + bootstrap; затем **`.env`** в `/opt/app` (скрипт `deploy/sweb/init-env-from-example.sh`).
- **Windows PowerShell `ssh-keygen -N ""`:** надёжнее интерактивно два раза Enter или `cmd /c "ssh-keygen ... -N \"\""`.

### Nginx / SSL (`deploy/sweb`)

- `nginx.conf` ожидает Let's Encrypt; до выпуска серта контейнер nginx может не подниматься; смок API: `docker compose exec -T api curl http://127.0.0.1:8000/health` (см. [docs/SWEB_RUNBOOK.md](../docs/SWEB_RUNBOOK.md)).

## 5. Этап 1 — модуль 2 на ноутбуке (публичный API)

**Цель:** убедиться, что без internal-режима работают health, metrics, process-image, скачивание, UI.

- **Переменные:** `INTERNAL_MODE=0`, рабочий **`OPENAI_API_KEY`** (при неверном ключе Vision даёт 401, возможен fallback в `gpt_agent` — проверять логи).
- **Зависимости тестов:** `pip install -r requirements-dev.txt`.
- **Pytest:** `tests/test_api_smoke.py`, `test_metrics_disk.py`, `test_upload_limit.py`, `test_all_image_types.py`, `test_product_background_mock.py`.
- **Ручной запуск:** `python -m uvicorn main:app --host 127.0.0.1 --port 8000` (если порт занят — `netstat -ano | findstr :8000`, `taskkill /PID … /F` или порт 8001).
- **Второй ПК Windows:** клон/копия проекта, venv, `requirements-dev.txt` или `requirements.txt`, перенос `.env` вручную, при доступе из LAN: `--host 0.0.0.0` + брандмауэр.
