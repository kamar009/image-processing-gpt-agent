# Описание четырёх функциональных модулей

*Логическое разбиение системы по зонам ответственности. Не совпадает один-к-одному с папками кода — это смысловые границы для обсуждения и планирования.*

---

## Модуль 1 — Движок обработки изображений

**Роль:** «мозг по работе с самой картинкой».

### Что делает

Принимает исходное фото и настройки сценария («как для товара», «как для баннера», **портфолио мебели** `furniture_portfolio` и т.д.). При необходимости **анализирует снимок через ИИ** (подключаемый провайдер: OpenAI, Sber, Yandex и др.), затем **последовательно применяет правила**: обрезка, фон, сжатие, целевой формат файла. В конце **проверяет результат** на типовые проблемы (например чрезмерная размытость или пересвет — по правилам проекта).

### Входные данные

| Тип | Содержание |
|-----|------------|
| Изображение | Байты или растровое представление после загрузки. |
| Параметры сценария | Тип картинки, режим фона, кроп, формат выхода, уровень качества и связанные настройки пресета. |
| Результат анализа ИИ (опционально) | Структурированные подсказки для геометрии и сценария, если этап Vision включён. |
| Настройки среды | Ограничение времени обработки, политика «предупреждения vs ошибки» при проверке качества и др. |

### Выходные данные

| Исход | Содержание |
|-------|------------|
| Успех | Готовый файл (байты), формат, размеры, краткий перечень выполненных операций, замеры времени по шагам. |
| Ошибка | Причина отказа (таймаут, валидация, сбой этапа) — на вызывающем уровне превращается в отказ API или статус «ошибка» у задачи в очереди. |

### Граница модуля

Не зависит от HTTP, Telegram, очередей и базы задач: только **«картинка + правила → результат или отказ»**.

### Связь с кодом (справочно)

`gpt_agent/`, `image_processor/`, `presets/`, `validator/`.

## Модуль 2 — Публичный API и синхронная выдача

**Роль:** «витрина для простого сценария: один запрос — один результат».

### Что делает

Принимает запросы извне (форма в браузере, интеграция партнёра), **в том же цикле запроса** запускает обработку и отдаёт результат. Обеспечивает проверки **живости сервиса** и **базовые метрики** для мониторинга. Подключает **общую веб-форму** и отдачу статики для неё.

### Входные данные

| Тип | Содержание |
|-----|------------|
| HTTP-запрос обработки | Файл изображения и поля сценария (тип, фон, формат и т.д.). |
| Служебные запросы | Запросы проверки работоспособности и метрик. |
| Браузер | Запросы к стартовой странице и статическим ресурсам формы. |

### Выходные данные

| Тип | Содержание |
|-----|------------|
| Результат обработки | Файл или структурированный ответ с описанием результата и ссылкой на скачивание (в терминах принятого API). |
| Диагностика | Ответы «сервис доступен / недоступен», агрегированные метрики. |
| Ошибки | Код и сообщение при невалидном вводе или сбое обработки. |

### Граница модуля

Может работать **без** внутренней очереди и без Telegram: только синхронная выдача и публичные точки входа.

### Связь с кодом (справочно)

Публичные маршруты и UI в `main.py`, локальное хранилище файлов для синхронного сценария — `output_storage/`, статика — `static/` (в т.ч. демо-форма `/ui` → `static/index.html`).

## Модуль 3 — Внутренняя платформа (очередь, доступ, данные)

**Роль:** «офис для внутренних пользователей: кто может, что в работе, что готово».

### Что делает

Включает **вход через Telegram**, **белый список пользователей**, выдачу **временных пропусков (токенов)** для безопасных запросов из браузера/Mini App. Задачи **ставятся в очередь**, сохраняются в базе; **фоновый обработчик** по очереди выполняет их, обновляя статусы: ожидание, в работе, готово, ошибка. Задаёт **лимиты** (размер файла, число одновременных задач на пользователя и т.д.). Администратор может **выдавать доступ** новым пользователям.

### Входные данные

| Тип | Содержание |
|-----|------------|
| Аутентификация Telegram | Данные инициализации Web App для проверки подлинности. |
| Создание и опрос задач | Идентификатор пользователя, ключ пресета, изображение (например в закодированном виде); запросы статуса и истории. |
| Защищённые запросы | Заголовок с выданным токеном доступа. |
| Настройки среды | Режим внутренней платформы, токен бота, списки администраторов, секрет подписи токенов, разрешённые источники для браузера, лимиты. |
| Для фонового обработчика | Записи задач в базе и файлы входа в хранилище по идентификаторам. |

### Выходные данные

| Тип | Содержание |
|-----|------------|
| Токены и ответы входа | Результат успешной или неуспешной аутентификации. |
| Задачи | Подтверждение постановки в очередь, список и детали статуса, сообщения об ошибках с пояснением. |
| Конфигурация для клиента | Лимиты и флаги для Mini App (размер загрузки, обязательность токена и т.д.). |
| Состояние здоровья внутреннего контура | Признаки доступности базы и записи в каталог результатов. |

### Граница модуля

Оркестрация **долгих** задач и **доверия к пользователю**; детали пикселей делегируются **модулю 1**.

### Связь с кодом (справочно)

`internal/`, фоновый процесс `worker.py`, префикс маршрутов `/internal/*` в `main.py`, вспомогательные скрипты в `scripts/` (синхронизация пресетов, проверки окружения и т.д.).

## Модуль 4 — Клиентские интерфейсы

**Роль:** «экраны для людей: форма в браузере и мини-приложение в Telegram».

### Что делает

- **Веб-форма** — загрузка файла, выбор сценария, отправка на **публичный** канал обработки.
- **Mini App в Telegram** — экран для сотрудников: загрузка, история задач, отображение ошибок сервера, подстройка под лимиты, запросы к **внутреннему** API.

### Входные данные

| Тип | Содержание |
|-----|------------|
| Действия пользователя | Выбор файла, сценария, отправка формы; в Mini App — контекст Telegram там, где он доступен клиенту. |
| Конфигурация с сервера | Лимиты и параметры отображения (например максимальный размер файла, нужен ли токен). |
| Ответы API | JSON с результатами, статусами задач, текстами ошибок. |

### Выходные данные

| Тип | Содержание |
|-----|------------|
| HTTP-запросы | К публичному API (модуль 2) или внутреннему API (модуль 3). |
| Представление для пользователя | Превью, списки, понятные сообщения об ошибках и успехе. |

### Граница модуля

Только **интерфейс и вызовы API**; бизнес-правила обработки изображений здесь не реализуются.

### Связь с кодом (справочно)

`static/index.html`, `static/miniapp/`.
